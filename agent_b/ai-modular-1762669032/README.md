# Freelance Job Auto-Applier Telegram Bot\n\nThis project implements a Telegram bot designed to help freelancers find and automatically apply for job listings across various platforms.\n\n## Features\n\n*   **Multi-Platform Scraping**: Gathers job listings from platforms like Upwork, Fiverr (planned), etc.\n*   **User Preferences**: Allows users to set keywords, blacklists, and other criteria via Telegram commands.\n*   **Automated Matching**: Matches scraped jobs against user preferences.\n*   **Auto-Applying**: Automatically applies to matched jobs based on predefined strategies (e.g., sending a generic cover letter, if API allows).\n*   **Persistent Storage**: Stores user settings, job listings, and application statuses using a database.\n*   **Scheduled Tasks**: Runs scraping and applying tasks on a regular schedule.\n\n## Project Structure\n\n\nfreelance_bot/\n├── app/                  # Application entry point and scheduler\n├── bot/                  # Telegram bot logic (handlers, keyboards)\n├── config/               # Configuration settings\n├── database/             # Database models, CRUD operations, session management\n├── scraper/              # Job scraping modules (base, platform-specific, manager)\n├── applier/              # Job application modules (base, platform-specific, manager)\n├── matcher/              # Job matching logic\n├── utils/                # Utility functions (logger, decorators)\n├── .env.example          # Example environment variables\n├── Dockerfile            # Docker configuration\n├── requirements.txt      # Python dependencies\n└── README.md             # This file\n\n\n## Setup\n\n### 1. Clone the repository\n\nbash\ngit clone https://github.com/your-username/freelance-bot.git\ncd freelance-bot\n\n\n### 2. Create a virtual environment and install dependencies\n\nbash\npython -m venv venv\nsource venv/bin/activate  # On Windows, use `venv\\Scripts\\activate`\npip install -r requirements.txt\n\n\n### 3. Environment Variables\n\nCreate a `.env` file in the root directory based on `.env.example` and fill in your details:\n\nini\n# .env\nBOT_TOKEN="YOUR_TELEGRAM_BOT_TOKEN"\nDATABASE_URL="sqlite:///./freelance_bot.db" # Or your PostgreSQL/MySQL URL\n\n# Upwork Credentials (example)\nUPWORK_EMAIL="your_upwork_email@example.com"\nUPWORK_PASSWORD="your_upwork_password"\n\n# Scheduling Intervals (in minutes)\nSCRAPER_INTERVAL_MINUTES=30\nAPPLIER_INTERVAL_MINUTES=60\n\n\n### 4. Database Initialization\n\nThe database schema will be created automatically on first run if it doesn't exist (for SQLite). For other databases, you might need a migration tool like Alembic.\n\n## Running the Bot\n\n### Locally\n\nbash\npython app/main.py\n\n\n### With Docker\n\nbash\ndocker build -t freelance-bot .\ndocker run --env-file ./.env freelance-bot\n\n\n## Usage\n\nStart a chat with your Telegram bot and use the following commands:\n\n*   `/start`: Welcome message and basic instructions.\n*   `/settings`: Configure your job preferences (keywords, blacklist, platforms, cover letter template).\n*   `/search`: Manually trigger a job search and see results.\n*   `/apply_status`: Check the status of your applications.\n\n## Contributing\n\nFeel free to fork the repository, make improvements, and submit pull requests.\n\n## License\n\nThis project is licensed under the MIT License - see the LICENSE file for details.
