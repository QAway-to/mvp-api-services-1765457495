# UI Design Guidelines for MVP Templates

## 🎯 Цель
Создание эргономичных, информативно-плотных интерфейсов для продающих MVP, которые демонстрируют 60-80% функциональности с минимальными пустыми пространствами.

## 📐 Эргономика и плотность информации

### Layout и Spacing
- **Используй grid layouts (2-3 колонки)** вместо вертикального списка для лучшего использования пространства
- **Минимизируй пустые пространства:**
  - Container padding: `16-20px` вместо `24-32px`
  - Card padding: `16px` вместо `24px`
  - Card margin-bottom: `12-16px` вместо `24px`
  - Gap между элементами: `10-12px` вместо `16-24px`

### Компактность
- **Заголовки:** `fontSize: 18-20px` вместо `24-36px`
- **Описания:** `fontSize: 12-14px` вместо `16px`
- **Карточки:** минимальная ширина `280px` для адаптивности
- **Максимальная ширина контейнера:** `1400px` с центрированием для больших экранов

## 📊 Визуализация данных

### Таблицы вместо списков
- Для **>5 элементов** используй таблицы вместо `<ul>` списков
- Таблицы должны быть:
  - Компактными (`fontSize: 12-13px`)
  - С hover эффектами на строках
  - С цветовым кодированием статусов
  - С пагинацией или ограничением видимых строк (20-50)

### Метрики и графики
- **Мини-графики:** progress bars для числовых метрик
- **Цветовое кодирование:**
  - Зелёный (`#22c55e`) — успех/положительные значения
  - Красный (`#ef4444`) — ошибки/отрицательные значения
  - Жёлтый (`#fbbf24`) — предупреждения/ожидание
  - Синий (`#38bdf8`) — информационные/нейтральные

### Компактные карточки метрик
```javascript
{
  background: '#0f172a',
  borderRadius: 10,
  padding: 12,
  // Включай progress bar для визуализации
}
```

## 🎨 Интерактивность

### Кнопки и действия
- **Все кнопки должны быть кликабельны** с визуальной обратной связью
- **Фиксированные размеры для кнопок:** кнопки (button элементы) ДОЛЖНЫ иметь фиксированную ширину (minWidth или width в пикселях)
- **Кнопки не должны менять размер:** после клика или hover кнопка должна сохранять свои размеры
- **Hover эффекты:** изменение opacity или background (БЕЗ изменения размеров)
- **Loading states:** для асинхронных операций
- **Disabled states:** визуально отличаются от активных
- **Остальные элементы адаптивные:** карточки, секции, таблицы могут быть адаптивными и менять размер в зависимости от контента

### Навигация
- Используй `<Link>` компоненты для внутренней навигации
- Внешние ссылки открывай в новой вкладке (`target="_blank" rel="noreferrer"`)

## 📱 Адаптивность

### Mobile-first подход
- **Responsive grid:** `gridTemplateColumns: 'repeat(auto-fit, minmax(280px, 1fr))'`
- **Flexible layouts:** используй `flexWrap: 'wrap'` для адаптации
- **Минимальная ширина карточек:** `280px`
- **Максимальная ширина контейнера:** `1400px` с `margin: '0 auto'`

### Breakpoints (для медиа-запросов, если нужны)
- Mobile: `< 640px` — одна колонка
- Tablet: `640px - 1024px` — 2 колонки
- Desktop: `> 1024px` — 3 колонки

## 🎭 Визуальные эффекты

### Анимации
- **Плавные переходы:** `transition: 'all 0.3s ease'` для интерактивных элементов
- **Анимация загрузки:** для ETL процессов показывай прогресс по шагам
- **Live updates:** обновляй данные в реальном времени с визуальной индикацией

### Цветовая схема (Dark theme)
- **Background:** `#0b1120` (основной), `#111c33` (карточки), `#0f172a` (вложенные)
- **Text:** `#f8fafc` (основной), `#e2e8f0` (вторичный), `#cbd5f5` (третичный), `#94a3b8` (описания)
- **Accents:** `#38bdf8` (primary), `#0ea5e9` (secondary), `#22c55e` (success), `#ef4444` (error)

## ✅ Чеклист для архитектора

При генерации UI компонентов убедись, что:
- [ ] Используется grid layout для метрик/карточек (2-3 колонки)
- [ ] Padding и margin минимизированы (16px вместо 24px)
- [ ] Таблицы используются для >5 элементов
- [ ] **Кнопки имеют фиксированные размеры** (minWidth или width в пикселях)
- [ ] **Кнопки не меняют размер при hover/click**
- [ ] Все кнопки кликабельны с hover эффектами (без изменения размеров)
- [ ] Метрики включают визуализацию (progress bars)
- [ ] Цветовое кодирование для статусов
- [ ] Адаптивный дизайн для карточек/секций (min-width: 280px, но могут быть адаптивными)
- [ ] Компактные заголовки (18-20px)
- [ ] Нет избыточных пустых пространств
- [ ] Карточки, секции, таблицы могут быть адаптивными (не требуют фиксированных размеров)

## 📚 Примеры

### Компактная метрика с progress bar
```javascript
function Metric({ label, value, max = 100 }) {
  const percentage = max > 0 ? Math.min((value / max) * 100, 100) : 0;
  return (
    <div style={{ background: '#0f172a', borderRadius: 10, padding: 12 }}>
      <p style={{ margin: 0, color: '#64748b', fontSize: 12 }}>{label}</p>
      <p style={{ margin: '6px 0', fontSize: 20, fontWeight: 700 }}>{value}</p>
      <div style={{ background: '#1e293b', borderRadius: 4, height: 4 }}>
        <div style={{ background: '#38bdf8', height: '100%', width: `${percentage}%` }} />
      </div>
    </div>
  );
}
```

### Компактная таблица
```javascript
<table style={{ width: '100%', borderCollapse: 'collapse', fontSize: 13 }}>
  <thead>
    <tr style={{ borderBottom: '1px solid rgba(148,163,184,0.2)' }}>
      <th style={{ padding: '8px 12px', fontSize: 11, textTransform: 'uppercase' }}>...</th>
    </tr>
  </thead>
  <tbody>
    {/* Строки с hover эффектами */}
  </tbody>
</table>
```

---

**Помни:** Цель — показать максимум функциональности в минимальном пространстве, создавая впечатление готового продукта на 60-80%.
